SMART on FHIR Examples

Introduction

As part of the NACHC hiv-cds project we are providing a series of example SMART on FHIR applications. These applications and the documentation here should enable a user to learn all of the basics to create a fully functional SMART on FHIR application. This tutorial assumes the user has a basic understanding of web development concepts including HTML, Javascript, etc.

The follow examples build up from the simplest possible SMART on FHIR application upto a fully functional SMART on FHIR application.

The link to each application will give instructions on how to deploy and test each SMART on FHIR application.

List of Example Applications

000-bootstrapping Start here! This page will walk through the steps to download, build, and deploy the example applications.
001-hello-world This is the simplest possible SMART on FHIR application. This application simply displays an HTML page that is launched from the EMR. This application does not authenticate to the EMR. This application does not get data from the EMR.
002-authenticate This is a basic SMART on FHIR application that adds the authentication step to the process. This application will authenticate to the EMR. This allows the EMR to forward the application from the initial launch (launch.html) page to a subsequent data (index.html) page. This application also uses FHIR to retrieve data from the EMR.
003-get-patient-data This SMART on FHIR application authenticates to the EMR and pulls and displays data from the EMR using FHIR. This is our first truly SMART on FHIR application as it uses both SMART and FHIR. This application includes several Javascript "alert" (popup) messages to demonstrate the flow of the application.
004-get-patient-data-quietly This SMART on FHIR application extends the 003-get-patient-data application by removing the Javascript alerts. This application authenticates to the EMR and pulls and displays patient data from the EMR.
